## Why

The project's own conventions (documented in `openspec/project.md`) mandate that all technical artifacts—including documentation—must be in English. However, key documentation files (`README.md` and `openspec/project.md`) contain Spanish text or mixed language content, creating inconsistency and violating the established language standards. This change aligns all technical documentation with the project's language conventions.

## What Changes

- Translate all Spanish text in `README.md` to English
- Translate all Spanish text in `openspec/project.md` to English
- Ensure section headers, body text, and instructions are consistently in English
- Preserve user-facing Spanish content references (since UI remains in Spanish per conventions)
- Update any mixed-language sections to use English exclusively for technical documentation

## Capabilities

### New Capabilities

None. This is a documentation content change only.

### Modified Capabilities

None. This change does not alter any functional requirements or capability specifications—it only standardizes the language of existing documentation to match project conventions.

## Impact

**Affected files:**
- `README.md` - Full translation of Spanish sections
- `openspec/project.md` - Full translation of Spanish sections

**Benefits:**
- Consistency with established project language conventions
- Improved accessibility for international contributors
- Clearer separation between technical documentation (English) and user interface (Spanish)
- Reduced language mixing confusion

**No code changes required** - this is purely documentation content standardization.
